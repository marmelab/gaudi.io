Microservices implies strong devops skills.
gaudi automates the setup of isolated and decoupled dev environments.

# Describe a SOA infrastructure 
# As a list of Docker containers
web_server:
    type: apache
    links: [web_app] # automate links between containers
    volumes:
        .: /var/www # mount code for static files
web_app:
    type: php-fpm
    links: [database, amqp]
    ports:
        9000: 9000 # Map network ports
    volumes:
        .: /var/www # mount code for php files
database:
    type: custom
    template: mysql/Dockerfile # Use your own Dockerfiles
    ports:
        3306: 3306
amqp:
    type: index # Use Docker containers from the public index
    image: toto/rabbitMQ
 
## If you're allergic to YAML, we also provide a GUI!

If you already use Docker to setup your dev box for large applications, don't you suffer to maintain complex Dockerfiles?

Splitting a service Oriented Architecture into several Docker containers is the solution, except managing links and mounts between them is a pain. gaudi solves that by offering a way to describe a system of Docker containers using a simple DSL.

Describe your infrastructure in seconds, run gaudi, and all the containers are up, running and connected.

Try our nice configuration file builder. See for yourself.

You can build your YAML configuration file directly in our webapp.This way, configuring your architecture is really intuitive.

gaudi is written in Go. It uses strong type-hinting and powerful asynchronism.

All features of gaudi are unit tested, these tests are executed after every feature addition.

gaudi is used by many developers and companies with various component types.
